Hugo ile Oluşturulan Bir Siteyi GitHub Pages ile Hostlamak

Merhaba, bugün sizlere Hugo ile oluşturduğunuz bir siteyi nasıl Github Pages ile hostlayabileceğinizi anlatacağım. Hugo, hızlı ve esnek bir statik site oluşturucusudur ve Github Pages ücretsiz olarak sitenizi barındırmanıza imkan verir.

Gerekli Araçlar ve Hazırlık Link to heading

  1. Hugo: Site oluşturucu.
  2. Git: Versiyon kontrol sistemi.
  3. Github hesabı: Github Pages’i kullanabilmek için bir Github hesabına ihtiyacınız var.
  4. Terminal: Komut satırı.

Adım 1: Hugo’yu Kurmak Link to heading

Öncelikle Hugo’yu kurmanız gerekiyor. Kurulum adımları kullandığınız işletim sistemine göre farklılık gösterebilir.

MacOS:

brew install hugo

Windows:

Hugo’yu Hugo Releases sayfasından indirip kurabilirsiniz.

Linux:

sudo apt-get install hugo

Kullandığınız Linux bağlı olarak kurulum komutları değişebilir.

Adım 2: Yeni Bir Hugo Projesi Oluşturmak Link to heading

Yeni bir Hugo projesi oluşturmak için aşağıdaki komutu çalıştırın:

hugo new site my-hugo-site

Bu komut, my-hugo-site adında yeni bir Hugo sitesi oluşturacaktır. Şimdi bu dizine geçiş yapalım:

cd my-hugo-site

Adım 3: Tema Ekleme Link to heading

Hugo’nun güçlü özelliklerinden biri de hazır temalarıdır. Hugo Themes sayfasından bir tema seçebilir ve ekleyebilirsiniz. Örnek olması açısından “Ananke” temasını ekleyelim:

git init

git submodule add https://github.com/theNewDynamic/gohugo-theme-ananke.git themes/ananke

Temayı hugo.toml dosyamıza ekleyelim:

theme = "ananke"

Adım 4: İçerik Oluşturmak Link to heading

Şimdi sitemize birkaç içerik ekleyelim. İlk yazımızı oluşturalım.

hugo new posts/ilk-yazi.md

Oluşturduğumuz dosya content/posts/ilk-yazi.md içerisinde bulunur. İstersek bu dosyayı açıp düzenleyebiliriz:

---
title: "İlk Yazı"
date: 2024-06-11T12:00:00
draft: true
---

Merhaba dünya! Bu benim ilk yazım.

draft: true kısmını draft: false olarak değiştirerek yazımızı yayına hazır hale getirelim.

Adım 5: Sitemizi Lokal Olarak Test Etmek Link to heading

Sitenizi lokal olarak görmek için şu komutu çalıştırabilirsiniz:

hugo server

Tarayıcınızda http://localhost:1313 adresine giderek sitenizi görebilirsiniz.

Adım 6: hugo.yaml Dosyasını Oluşturmak Link to heading

Sitenizin bulunduğu klasöre hugo.yaml dosyası oluşturun: .github/workflows/hugo.yaml Oluşturduğunuz dosyaya aşağıdaki komutu yapıştırın:

# Sample workflow for building and deploying a Hugo site to GitHub Pages
name: Deploy Hugo site to Pages

on:
 # Runs on pushes targeting the default branch
 push:
   branches:
     - main

 # Allows you to run this workflow manually from the Actions tab
 workflow_dispatch:

# Sets permissions of the GITHUB_TOKEN to allow deployment to GitHub Pages
permissions:
 contents: read
 pages: write
 id-token: write

# Allow only one concurrent deployment, skipping runs queued between the run in-progress and latest queued.
# However, do NOT cancel in-progress runs as we want to allow these production deployments to complete.
concurrency:
 group: "pages"
 cancel-in-progress: false

# Default to bash
defaults:
 run:
   shell: bash

jobs:
 # Build job
 build:
   runs-on: ubuntu-latest
   env:
     HUGO_VERSION: 0.126.0
   steps:
     - name: Install Hugo CLI
       run: |
         wget -O ${{ runner.temp }}/hugo.deb https://github.com/gohugoio/hugo/releases/download/v${HUGO_VERSION}/hugo_extended_${HUGO_VERSION}_linux-amd64.deb \
         && sudo dpkg -i ${{ runner.temp }}/hugo.deb          
     - name: Install Dart Sass
       run: sudo snap install dart-sass
     - name: Checkout
       uses: actions/checkout@v4
       with:
         submodules: recursive
         fetch-depth: 0
     - name: Setup Pages
       id: pages
       uses: actions/configure-pages@v4
     - name: Install Node.js dependencies
       run: "[[ -f package-lock.json || -f npm-shrinkwrap.json ]] && npm ci || true"
     - name: Build with Hugo
       env:
         # For maximum backward compatibility with Hugo modules
         HUGO_ENVIRONMENT: production
         HUGO_ENV: production
         TZ: America/Los_Angeles
       run: |
         hugo \
           --gc \
           --minify \
           --baseURL "${{ steps.pages.outputs.base_url }}/"          
     - name: Upload artifact
       uses: actions/upload-pages-artifact@v3
       with:
         path: ./public

 # Deployment job
 deploy:
   environment:
     name: github-pages
     url: ${{ steps.deployment.outputs.page_url }}
   runs-on: ubuntu-latest
   needs: build
   steps:
     - name: Deploy to GitHub Pages
       id: deployment
       uses: actions/deploy-pages@v4

Branch ismini ve Hugo sürümünü kendinize uygun şekilde değiştirin.

Adım 7: Github Pages Yapılandırması Link to heading 

git remote add origin https://github.com/kullaniciadi/kullaniciadi.github.io.git
git add .
git commit -m "İlk gönderi"
git push -u origin master

Adım 8: Sitemizi Üretmek ve Yayınlamak Link to heading

Sitemini statik dosyalara dönüştürmek için aşağıdaki komutu çalıştırın:

hugo

Bu komut, public adlı bir dizin oluşturacaktır. Şimdi bu dizini GitHub Pages’e itmek için bazı ek adımlar atmamız gerekiyor. public dizinini ayrı bir branch (örneğin gh-pages) olarak yönetebiliriz.

cd public
git init
git remote add origin https://github.com/kullaniciadi/kullaniciadi.github.io.git
git checkout -b gh-pages
git add .
git commit -m "Site yayında"
git push -u origin gh-pages

Adım 9: Github Pages Ayarları Link to heading

GitHub üzerindeki depoya gidin ve Settings > Pages sekmesine tıklayın. Burada, gh-pages branch’ini seçin ve kaydedin. Birkaç dakika sonra siteniz https://kullaniciadi.github.io adresinde yayında olacaktır.
